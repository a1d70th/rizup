# セキュリティポリシー

最終更新日：2026年4月7日

## 1. 基本方針

株式会社 Rizup は、ユーザーの個人情報とデータを適切に保護することを最優先事項とし、以下のセキュリティ対策を実施します。

## 2. インフラセキュリティ

### ホスティング・データベース
- **Vercel**（アプリケーションホスティング）：SOC 2 Type II 準拠
- **Supabase**（データベース・認証）：SOC 2 Type II 準拠、データ暗号化（AES-256）
- **Stripe**（決済処理）：PCI DSS Level 1 準拠

### 通信の暗号化
- すべての通信を SSL/TLS（HTTPS）で暗号化
- API エンドポイントは TLS 1.2 以上を要求
- HSTS（HTTP Strict Transport Security）を有効化

### データの暗号化
- データベース内の個人情報は暗号化して保存（AES-256）
- バックアップデータも暗号化
- クレジットカード情報は当社サーバーに一切保存しない（Stripe が管理）

## 3. 認証・アクセス制御

### ユーザー認証
- Supabase Auth によるセキュアな認証
- パスワードは bcrypt でハッシュ化して保存
- メール確認フローによるアカウント検証
- セッショントークンは JWT（有効期限付き）

### アクセス制御
- Row Level Security（RLS）により、ユーザーは自分のデータのみアクセス可能
- 管理者アクセスは最小権限の原則に基づく
- API キーは環境変数として管理（ソースコードに含めない）

## 4. アプリケーションセキュリティ

### 入力バリデーション
- すべてのユーザー入力をサーバー側でバリデーション
- SQL インジェクション対策（パラメータ化クエリ）
- XSS 対策（出力エスケープ・Content Security Policy）
- CSRF 対策（トークンベース）

### API セキュリティ
- レートリミットの設定（DoS 攻撃防止）
- 認証トークンの検証をすべてのエンドポイントで実施
- エラーメッセージに内部情報を含めない

### 依存パッケージ
- 定期的な依存パッケージの脆弱性スキャン
- 重大な脆弱性が発見された場合は24時間以内にアップデート

## 5. AI データ処理のセキュリティ

- Claude API への送信データには個人識別情報（メール・本名）を含めない
- 送信データは投稿テキストと気分データのみに限定
- Anthropic の利用規約に基づき、API経由のデータはモデルの学習に使用されない
- AI 処理結果は当社データベースに保存し、第三者に提供しない

## 6. コンテンツモデレーション

- 投稿送信前に Claude API でリアルタイムスキャン
- 個人情報（電話番号・住所・メールアドレス）の自動検出・ブロック
- 自傷・自殺関連ワードの検出時は即座に相談窓口を表示
- 詳細は `app/moderation-spec.md` を参照

## 7. インシデント対応

### 対応フロー
1. インシデント検知（自動監視 or ユーザー報告）
2. 影響範囲の特定（1時間以内）
3. 一次対応（被害拡大の防止）
4. ユーザーへの通知（個人情報漏洩の場合は72時間以内）
5. 原因調査・恒久対策
6. 再発防止策の実施

### 報告義務
- 個人情報の漏洩が発生した場合、個人情報保護委員会に報告
- 影響を受けるユーザーに速やかに通知
