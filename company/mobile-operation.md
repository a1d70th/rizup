## 1. スマホからClaude.aiで会社を運営する方法

### 基本セットアップ

1. **Claude.ai（https://claude.ai）** にスマホブラウザまたはClaudeアプリからアクセス
2. プロジェクト機能を使い「Rizup 運営」プロジェクトを作成
3. プロジェクトの「Instructions」に `CLAUDE.md` の内容を貼り付ける
4. これにより、毎回の会話でAI社員の役割定義・連携ルールが自動適用される

### 運営の基本フロー

```
社長（スマホ）→ Claude.ai に指示 → AI社員として作業 → 成果物を確認 → 承認/修正指示
```

### ポイント

- Claude.ai のプロジェクト機能で「会社のコンテキスト」を常に保持できる
- 1つの会話で複数の社員に順番に指示を出せる
- 長い会話は新規チャットに切り替え、プロジェクト指示が引き継がれる

## 2. Claude Code (CLI) vs Claude.ai (Web) の使い分け

| 項目 | Claude Code (CLI) | Claude.ai (Web/アプリ) |
|---|---|---|
| **デバイス** | PC（ターミナル必須） | スマホ/タブレット/PC |
| **得意なこと** | コード編集、git操作、ファイル直接操作 | 戦略検討、文章作成、指示出し |
| **MCP連携** | Notion/GitHub/Calendar 直接操作可能 | プロジェクト指示で疑似的に対応 |
| **適した作業** | Soraへの開発タスク、デプロイ確認 | 朝礼、日報確認、戦略会議、営業文作成 |
| **移動中** | 使用不可 | 使用可能 |

### 使い分けルール

- **スマホ（Claude.ai）**: 指示出し、レビュー、戦略・マーケ・営業系タスク
- **PC（Claude Code）**: コード変更、git push、Notion/GitHub直接操作、DB作業
- **判断基準**: 「ファイルを直接触る必要があるか？」→ Yes なら Claude Code、No なら Claude.ai

## 3. スマホでの指示テンプレート

### 基本テンプレート（コピペ用）

```
@Haru として動いてください。

■ タスク：
■ 背景・目的：
■ 完了条件：
■ 期限：
■ 依存タスク：
■ 出力先：
```

### 短縮テンプレート（移動中用）

スマホでは簡潔な指示でも、プロジェクト指示があるため正しく動作する。

```
@Haru 今日のタスク割り振りをお願い。優先度は①LP改善②note記事③営業提案文。
```

```
@Sora LPのCTAボタンの色を#6ecbb0に変更して、クリック率改善を狙いたい。
```

```
@Kai コミュニティ有料プランの価格設定について、競合3社の調査をお願い。
```

```
@Rei 今週のX投稿を5本作成。テーマは「転職に悩む20代」向け。
```

```
@Leo クラウドワークスでLP制作案件を3件リサーチ。予算5万円以上のもの。
```

### 複数社員への一括指示

```
本日の全社指示です。各社員として順番に動いてください。

@Haru: タスクボード更新。昨日の進捗をまとめて。
@Sora: Issue #12 のバグ修正の仕様をまとめて。
@Kai: 今週のKPIレポート作成。
@Rei: 明日のX投稿3本を下書き。
@Leo: 進行中の案件ステータス報告。
```

## 4. 朝礼コマンド

毎朝スマホから送る定型指示。コピペして使う。

### 標準朝礼コマンド

```
おはようございます。朝礼を始めます。

@Haru として動いてください。

■ タスク：本日の朝礼実施
■ 背景・目的：全社員の業務方針を確定し、1日を効率的に進める
■ 完了条件：以下の項目をすべてアウトプットすること
  1. 昨日の成果サマリー（各社員）
  2. 本日の優先タスク（各社員に割り振り）
  3. 月間売上目標（30万円）に対する進捗率
  4. ブロッカーや懸念事項
  5. 社長への確認・承認依頼事項
■ 期限：即時
■ 出力先：この会話内
```

### 簡易朝礼（忙しい朝用）

```
@Haru 朝礼。昨日の振り返りと今日の最優先タスクを各社員分お願い。30万円目標の進捗も。
```

### 週初め朝礼（月曜日用）

```
@Haru 週次朝礼をお願いします。

1. 先週の成果まとめ（各社員）
2. 先週のKPI実績 vs 目標
3. 今週の重点施策（各社員の週間タスク）
4. 30万円目標への今月の見通し
5. リスクと対策
```

## 6. 緊急対応フロー

### サービス障害時

```
【緊急】@Sora として動いてください。

■ タスク：サービス障害の調査と対応
■ 状況：（障害の内容を記載）
■ 完了条件：
  1. 原因の特定
  2. 修正方針の提示
  3. 修正コードの作成（Claude Code で実行する前提）
■ 期限：即時
```

### クライアント緊急対応

```
【緊急】@Leo として動いてください。

■ タスク：クライアントへの緊急返信文作成
■ 状況：（クライアントからの連絡内容を記載）
■ 完了条件：返信文のドラフト作成（社長確認後に送信）
■ 期限：30分以内
■ 注意：クライアント情報は含めない。文面のみ作成。
```

### 炎上・クレーム対応

```
【緊急】@Haru 緊急対応の割り振りをお願いします。

■ 状況：（問題の概要）
■ 影響範囲：（どの事業に影響があるか）
■ 必要な対応：
  1. 初動対応（誰が何をするか）
  2. 根本原因調査（誰が担当するか）
  3. 再発防止策の検討
```

## 7. Notion連携（MCP確認方法）

### 前提

- MCPサーバー（notion-mcp）は Claude Code (CLI) 経由でのみ直接操作可能
- スマホからはClaude.aiで「指示を準備」→ PCに戻ったときにClaude Codeで実行

### スマホからの確認依頼テンプレート

```
@Haru 以下のNotion確認タスクをまとめてください。PCに戻ったらClaude Codeで実行します。

1. 今週のタスクボードの進捗確認
2. KPIダッシュボードの数値更新
3. 議事録ページの作成（本日分）
```

### PCに戻ったときの実行コマンド（Claude Code用メモ）

```
Notionのタスクボードを確認して、今週の進捗をサマリーしてください。
```

```
NotionのKPIページを更新してください。今月の実績：
- クラウドワークス: ¥XX,XXX
- note: ¥XX,XXX
- アフィリエイト: ¥XX,XXX
```

## 8. GitHub Issue/PR確認方法

### スマホからの確認（Claude.ai）

GitHub MCPもClaude Code経由のため、スマホからは以下のように確認指示を準備する。

```
@Sora 以下のGitHub確認をまとめてください：

1. オープン中のIssue一覧と優先度
2. 未マージのPR一覧とレビュー状況
3. 直近のデプロイ状況（最新コミット）
```

### PCでの実行用メモ

```
# Issue確認
GitHubのa1d70th/rizupリポジトリのオープンIssueを一覧表示してください。

# PR確認
オープン中のPRを確認して、マージ可能かレビューしてください。

# デプロイ確認
最新のコミットログを5件表示してください。
```

### スマホのブラウザから直接確認

- Issues: https://github.com/a1d70th/rizup/issues
- PRs: https://github.com/a1d70th/rizup/pulls
- Actions: https://github.com/a1d70th/rizup/actions

## 10. Tips: 効率化テクニック

### 音声入力の活用

スマホの最大の武器は音声入力。長文の指示も声で素早く入力できる。

- **iOS**: キーボードのマイクボタンを長押し
- **Android**: Gboard のマイクアイコンをタップ
- **コツ**: 「あっとまーく はる として うごいてください」→ 送信前に「@Haru」に手動修正

**音声入力向きのタスク：**
- 朝礼の追加指示
- フィードバックコメント
- 戦略アイデアのメモ
- クライアント対応方針の口述

### テンプレート保存方法

#### iPhoneの場合
1. **ユーザ辞書登録**: 設定 > 一般 > キーボード > ユーザ辞書
   - 「あされい」→ 朝礼コマンド全文
   - 「にっぽう」→ 日報依頼テンプレート
   - 「はる」→ `@Haru として動いてください。`
   - 「そら」→ `@Sora として動いてください。`

2. **ショートカットApp**: 朝礼テンプレートをクリップボードにコピーするショートカットを作成

3. **メモApp**: テンプレート集をメモに保存し、必要時にコピペ

#### Androidの場合
1. **Gboard クリップボード**: よく使うテンプレートをピン留め
2. **Google Keep**: テンプレート集を保存

### Claude.ai のショートカット

- **プロジェクト機能**: CLAUDE.mdの内容をプロジェクトに設定すれば、毎回貼り付け不要
- **会話の切り替え**: 長くなったら新規チャットへ（プロジェクト指示は自動引き継ぎ）
- **アーティファクト**: Claudeが生成した文書はアーティファクトとして保存・再利用可能

### スマホ運用のベストプラクティス

1. **指示は短く、明確に** — スマホでは長文入力が面倒。要点だけ伝える
2. **確認と承認に集中** — 作業はAI社員に任せ、社長はレビューに徹する
3. **PCとの連携を意識** — スマホで指示準備 → PCで実行のフローを作る
4. **通知を活用** — Claude.aiアプリの通知をONにして応答を見逃さない
5. **1チャット1テーマ** — 話題が変わったら新しいチャットを開始する

### よく使うフレーズ集

| フレーズ | 用途 |
|---|---|
| `@Haru 進捗報告` | 全社の進捗確認 |
| `@Sora ビルド状況確認` | デプロイ状況の確認 |
| `@Kai 数字で教えて` | データに基づく回答を要求 |
| `@Rei 投稿案3本` | X投稿の下書き依頼 |
| `@Leo 案件リサーチ` | クラウドワークス案件の調査 |
| `承認します` | AI社員の成果物を承認 |
| `修正してください：` | 成果物の修正指示 |
| `優先度を変更：` | タスクの優先度変更 |

## 補足: セキュリティ注意事項

- クライアントの個人情報や機密情報はClaude.aiに入力しない
- パスワード・APIキーなどの認証情報は絶対に送信しない
- クライアント向け文面は必ず社長が最終確認してから送信する
- 公共Wi-Fiでの作業時はVPNの使用を推奨する
