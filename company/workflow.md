## ワークフロー 1：クラウドワークス案件の受注〜納品

**関係者：Leo → Kai → Sora or Rei → Haru**

```
Step 1 │ Leo    │ クラウドワークスで案件をリサーチ
       │        │ → /consulting/pipeline.md に候補を記録
       │        │
Step 2 │ Kai    │ 案件の相場・競合提案を調査
       │        │ → Leo に適正価格と差別化ポイントを伝える
       │        │
Step 3 │ Leo    │ 提案文を作成
       │        │ → /consulting/proposals/ に保存
       │        │ → 社長が確認・承認
       │        │
Step 4 │ Leo    │ 提案を送信 → 受注
       │        │ → Haru にタスク発行を依頼
       │        │
Step 5 │ Haru   │ 案件をタスクに分解し、担当を割り振る
       │        │  ├ LP制作案件 → Sora
       │        │  ├ 記事案件   → Rei
       │        │  └ 広告案件   → Kai + Rei
       │        │
Step 6 │ Sora   │ 制作・開発を実行
   or  │ Rei    │ → 成果物を GitHub にコミット
       │        │
Step 7 │ Leo    │ 納品物を確認 → クライアントに納品
       │        │ → 社長が最終チェック
       │        │
Step 8 │ Haru   │ 売上を記録・KPI を更新
       │        │ → /company/daily-report.md に記載
```

## ワークフロー 2：note 記事の企画〜公開〜収益化

**関係者：Kai → Rei → Sora → Haru**

```

       │        │

       │        │
Step 3 │ Rei    │ 記事本文を執筆
       │        │ → アフィリリンクを適切に配置
       │        │ → CTA にRizupコミュニティ導線を設置
       │        │
Step 4 │ 社長   │ 記事を最終レビュー → note に公開
       │        │
Step 5 │ Rei    │ X / Instagram で記事を宣伝
       │        │ → /media/x-posts.md に投稿文を記録
       │        │
Step 6 │ Sora   │ LP に記事へのリンクを追加（必要に応じて）
       │        │
Step 7 │ Haru   │ PV・売上を記録・KPI を更新
```

## 部門間連携のルール

### 依頼の流れ

```
社員A が 社員B に作業を依頼したい場合：

  ❌ 社員A → 社員B（直接依頼は禁止）
  ✅ 社員A → Haru → 社員B（必ず Haru 経由）
```

**理由：** Haru がすべてのタスクを把握することで、優先順位の衝突や抜け漏れを防ぐ。

---

### 依存関係マトリクス

どの社員が誰の成果物を必要とするかの一覧。

| ← 依頼先 / 依頼元 → | Haru | Sora | Kai | Rei | Leo |
|---|---|---|---|---|---|
| **Haru** | - | 進捗報告 | 調査結果 | 成果報告 | 受注報告 |
| **Sora** | タスク指示 | - | 改善提案 | コピー文 | 案件仕様 |
| **Kai** | 調査依頼 | 技術的制約 | - | テーマ相談 | 相場依頼 |
| **Rei** | タスク指示 | LP更新通知 | 調査データ | - | 案件要件 |
| **Leo** | タスク指示 | 納品物 | 相場データ | クリエイティブ | - |

---

### 情報共有ルール

| 情報 | 共有先 | 共有方法 |
|---|---|---|
| 日次の進捗 | 全員 | Haru が日報にまとめる |
| 売上数字 | 全員 | Haru がKPIダッシュボードを更新 |
| クライアント情報 | Leo + 担当者のみ | 案件ファイルに記載 |
| 調査レポート | 関係者 | /consulting/ に保存 → Haru が共有 |
| コンテンツ | 全員 | /media/ に保存 → Haru が共有 |

## 1日の流れ（理想形）

```
09:00  社長      朝礼コマンドを打つ
09:05  Haru      タスクを各社員に割り振る
09:10  全社員    各自のタスクに取り掛かる
       │
       ├─ Sora   LP改善 or 案件開発
       ├─ Kai    調査 or 分析
       ├─ Rei    コンテンツ作成
       └─ Leo    案件リサーチ or 提案文作成
       │
12:00  社長      午前の成果物を確認・フィードバック
       │
       ├─ 各社員  フィードバックをもとに修正
       │
17:00  各社員    完了報告を Haru に提出
17:10  Haru      日報作成・KPI更新・翌日の準備
17:30  社長      日報を確認・翌日の優先事項を決める
```
