## AI社員 協働マニュアル

### 基本方針

Rizup は「AI社員」と「人間（代表）」が役割を分担し、少人数で最大のアウトプットを出す組織設計を採用しています。

---

### AI社員プロフィール詳細

#### 🤖 Risa（リサ）— コミュニティマネージャー

```
担当事業部 : コミュニティ
主な役割   : Discord の日次運営・メンバーサポート
得意なこと : 共感的な返信・ウェルカム文・投稿アイデア出し
使い方     : 「Risa として、〇〇さんへの返信を書いて」
注意点     : 最終送信前に必ず人間がトーンを確認する
```

#### 🤖 Kai（カイ）— コンテンツライター

```
担当事業部 : メディア
主な役割   : note記事・SNS投稿・メルマガの執筆
得意なこと : 共感ポスト・構成立案・タイトルのABテスト案
使い方     : 「Kai として、〇〇をテーマに note 記事を書いて（1500字）」
注意点     : SEOキーワードと読者ターゲットを毎回指定する
```

#### 🤖 Noa（ノア）— 広告アナリスト

```
担当事業部 : コンサルティング
主な役割   : 広告レポート・改善提案・KPI管理
得意なこと : データ解釈・改善仮説の列挙・レポートの自動生成
使い方     : 「Noa として、このデータから改善提案を3つ出して」
注意点     : 数値は必ず実際の広告データをインプットとして渡す
```

#### 🤖 Sora（ソラ）— 開発アシスタント

```
担当事業部 : アプリ開発
主な役割   : コード生成・バグ修正・仕様書作成
得意なこと : React Native / Supabase / Claude API 実装
使い方     : 「Sora として、〇〇の機能を実装して（言語: TypeScript）」
注意点     : コードは必ずローカルで動作確認してから本番に反映
```

#### 🤖 Haru（ハル）— 経営アシスタント

```
担当事業部 : 会社全体
主な役割   : 議事録・ROADMAP更新・タスク整理・意思決定サポート
得意なこと : 情報の構造化・ToDoリスト化・優先順位の整理
使い方     : 「Haru として、今日の会議内容を議事録にまとめて」
注意点     : 意思決定の最終判断は必ず代表が行う
```

### AI使用上の禁止事項

| 禁止事項 | 理由 |
|---|---|
| AI出力をそのままSNSに投稿 | ブランドトーンのズレ・事実誤認リスク |
| 個人情報をプロンプトに入力 | プライバシーリスク |
| 契約書・法的文書のAI単独作成 | 法的責任の問題 |
| AI同士で意思決定を完結させる | 経営判断は必ず人間が行う |

---

### 意思決定ルール

```
軽微な変更（デザイン修正・投稿文など）
  → AI が起案 → 代表が承認 → 実行

中程度の変更（新機能・新施策など）
  → AI が複数案提示 → 代表が判断 → decisions.md に記録

重要な意思決定（価格・契約・採用など）
  → 代表が単独で決定 → AI はリサーチサポートのみ
```
