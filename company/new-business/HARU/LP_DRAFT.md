## 🌐 言語切り替え設計（双方向 LP）

### 基本仕様
- **URL**: `haru-osaka.vercel.app`（仮） or Rizup 内の `/haru` サブパス
- **ページ上部に切替ボタン**: `🇯🇵 日本語 | 🇰🇷 한국어`（右上に固定）
- **デフォルト言語の自動判定**:
  1. URL クエリパラメータ `?lang=ko` or `?lang=ja` 最優先
  2. localStorage `haru_lang` の保存値
  3. ブラウザの `navigator.language`（`ja-*` → 日本語 / `ko-*` → 韓国語 / その他 → 日本語）
- **切替時の挙動**: ページリロードなしで CSS `display` + `lang` 属性切替。URL に `?lang=xx` を push state で残す（シェア時に言語維持）

## 🎨 ビジュアル方針

### カラーパレット
- **Primary**: #FFB7C5（桜色）
- **Accent**: #FF6B9D（濃いピンク）
- **Text**: #2D2D2D（墨）
- **Background**: #FFF5F7（薄桜 / ベースBG）
- **Subtle**: #FFE5EB（カード背景）

### フォント
- **日本語**: Noto Sans JP + Zen Maru Gothic（見出し）
- **韓国語**: Noto Sans KR（全体）/ Gothic A1（見出しのバリエーション）
- **数字 / 英語**: Space Mono or DM Mono（日付・時刻等）

### 装飾
- 桜の花びらパーティクル（軽量・控えめ）
- 角丸・shadow・pastel 全体
- Dark mode は **未対応**（コミュニティ温度感を優先）

## 🔧 実装方針（段階的）

### Phase 1（今週・最速 MVP）
- **単一 HTML ファイル** で LP 作成（Next.js 不要・Vercel に static deploy）
- JavaScript で言語切替トグル（CSS `.lang-ja` / `.lang-ko` クラスで表示制御）
- Instagram 埋込は iframe
- 合計 1 ファイル 400 行以内で完結させる

### Phase 2（集客安定後）
- Rizup 本体に `/haru` ルートとして統合（Next.js ページ化）
- Instagram API でフィード動的取得
- Supabase `haru_events` テーブルから次回開催情報を読み込み

### Phase 3（事業化したら）
- 独自ドメイン `haru-osaka.com` 取得
- Japan + Korea の SEO 対策
- 多言語 OG タグ（og:locale:alternate）
