# 朝礼コマンド テンプレート

> 毎朝、社長が Claude Code で打つ一言で、全社員の1日が動き出す。

---

## 基本の朝礼コマンド

毎朝これをコピペして、`{}`内を書き換えて実行してください。

```
@Haru として動いてください。朝礼を開始します。

■ 今日の日付：{2026年X月X日（曜日）}
■ 社長の優先事項：{今日最も進めたいこと}
■ 昨日の振り返り：{昨日の成果・気づき}
■ 今日の売上目標：¥{金額}
■ ブロッカー：{今止まっていること・困っていること}

以下を実行してください：
1. 昨日の各社員の進捗を確認し、サマリーを出す
2. 今日の全社員のタスクを割り振る（担当・完了条件・期限つき）
3. 30万円達成までの進捗率を計算する
4. 今日の最優先アクション TOP3 を提示する
5. /company/daily-report.md に今日のタスク表を書き込む
```

## 曜日別の朝礼コマンド

### 月曜日（週次スタート）

```
@Haru として動いてください。月曜朝礼を開始します。

■ 今日の日付：{2026年X月X日（月）}
■ 今週の目標：{今週達成したいこと}
■ 先週の売上実績：¥{金額}
■ 30万円まで残り：¥{金額}

以下を実行してください：
1. 先週の各社員の成果をサマリーにまとめる
2. 先週のKPI進捗を一覧で出す
3. 今週の全社員のタスクを週次計画として割り振る
4. 今週のクリティカルパス（最も重要な依存タスク）を特定する
5. /company/weekly-plan.md に今週の計画を書き込む
```

### 金曜日（週次クロージング）

```
@Haru として動いてください。金曜クロージングを開始します。

■ 今日の日付：{2026年X月X日（金）}
■ 今週の振り返り：{うまくいったこと・課題}

以下を実行してください：
1. 今週の各社員の成果を集計する
2. KPI の週次進捗を更新する
3. 今週の売上・成果を数値でまとめる
4. 来週に持ち越すタスクをリスト化する
5. /company/weekly-report.md に週次レポートを書き込む
```

## 特定社員への直接指示コマンド

### Sora（開発）に LP 改善を指示

```
@Sora として動いてください。

■ タスク：LP の CVR 改善
■ 背景：{Kai の調査で〇〇が判明したため}
■ 変更内容：{具体的な変更箇所}
■ 完了条件：コード変更 → commit → push まで完了
■ 期限：今日中
```

### Kai（リサーチ）に市場調査を依頼

```
@Kai として動いてください。

■ タスク：{調査テーマ}
■ 背景：{なぜこの調査が必要か}
■ 調査観点：
  - 市場規模
  - 競合サービス3つ以上
  - 価格帯の相場
  - Rizup が狙うべきポジション
■ 出力形式：調査レポート（事実→分析→提案3案）
■ 出力先：/consulting/market-research.md
```

### Rei（マーケ）にコンテンツ作成を依頼

```
@Rei として動いてください。

■ タスク：{コンテンツ種別（X投稿 / note記事 / etc.）}
■ テーマ：{テーマ}
■ ターゲット読者：{誰に向けて}
■ トーン：親しみやすい・共感的
■ 文字数：{X投稿: 140字以内 / note: 2000字 / etc.}
■ CTA：{読者にしてほしいアクション}
■ 出力先：/media/x-posts.md or /media/note-drafts/
```

### Leo（営業）に案件獲得を指示

```
@Leo として動いてください。

■ タスク：クラウドワークス案件リサーチ & 提案文作成
■ 案件ジャンル：{LP制作 / ライティング / 広告運用 / etc.}
■ 予算帯：{3万〜10万円}
■ 提案文に含めること：
  - 相手の課題への共感
  - Rizup としての解決策
  - 想定スケジュール
  - 価格
■ 出力先：/consulting/proposals/
■ 注意：提案文は社長確認後に送信
```

## 緊急コマンド

### 売上が足りないとき

```
@Haru として動いてください。緊急売上会議を開催します。

■ 今月の売上実績：¥{金額}
■ 今月の目標：¥300,000
■ 残り日数：{X日}
■ 不足額：¥{金額}

以下を実行してください：
1. 不足額を埋めるための施策を各社員から3案ずつ出す
   - Leo：即受注可能な案件候補
   - Rei：すぐ収益化できるコンテンツ案
   - Kai：未開拓の収益チャネル調査
   - Sora：LP改善で CVR を上げる施策
2. 全施策を「実現速度×インパクト」でランク付けする
3. 今週やるべきTOP3を決定する
```

### トラブル発生時

```
@Haru として動いてください。

■ トラブル内容：{何が起きたか}
■ 影響範囲：{誰に・何に影響があるか}
■ 緊急度：高 / 中 / 低

以下を実行してください：
1. トラブルの原因を特定する
2. 最も早い解決策を提示する
3. 担当社員にタスクを割り振る
4. 再発防止策を提案する
```

## Tips：朝礼を習慣化するコツ

1. **毎朝同じ時間に打つ**（例：9:00）
2. **完璧に書かなくていい** — 一言でもOK。Haru が補完してくれる
3. **昨日の振り返りから書く** — 何をしたか思い出すと今日やることが見える
4. **売上数字は毎日見る** — 数字を意識するだけで行動が変わる
