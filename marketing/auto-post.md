## 全体フロー概要

```
┌─────────────┐    ┌──────────────┐    ┌──────────────┐    ┌─────────────┐
│  テーマ選定  │ →  │ Claude AI で  │ →  │  社長承認     │ →  │  手動投稿    │
│  (Rei)      │    │  下書き生成   │    │  (Shohei)    │    │  (Rei)      │
└─────────────┘    └──────────────┘    └──────────────┘    └─────────────┘
       ↑                                      │
       │                                      ↓
  Kai のリサーチ結果                    修正依頼（あれば）
  コンテンツカレンダー                  → Claude で再生成
```

**原則:**
- AI社員の出力をそのまま外部に公開しない（社長レビュー必須）
- 投稿バランス: **共感7 : 情報2 : 宣伝1**
- 全投稿に Rizup コミュニティへの自然な導線を意識する

## X投稿の半自動化フロー

### ステップ1: テーマ決定

Kai のリサーチ結果とコンテンツカレンダー（`/media/content-calendar.md`）を確認し、当日のテーマを決定する。

**テーマ選定の判断軸:**
- トレンドとの親和性（X のトレンドを確認）
- ターゲット層の関心事（20〜30代、キャリアに悩む層）
- Rizup のミッションとの整合性

### ステップ2: Claude AI で下書き生成

以下のプロンプトテンプレートを使って Claude に下書きを依頼する。

```
あなたは株式会社Rizupのマーケター「Rei」です。

■ ブランドボイス:
- 温かく、前向きで、押しつけがましくない
- 「頑張れ」ではなく「一緒にやろう」のトーン
- 専門用語を避け、日常的な言葉を使う

■ ターゲット:
- 20〜30代、今の自分に自信が持てない人
- 転職・副業・自己成長に興味がある層

■ 投稿タイプ: {共感 / 情報 / 宣伝}
■ テーマ: {具体的なテーマ}
■ 文字数: 140文字以内（X制限）
■ 含めたい要素: {あれば}

上記の条件でX投稿の下書きを3パターン作成してください。
```

### ステップ3: 社長承認

- 下書き3パターンを `/media/x-posts.md` に追記
- 社長が確認し、採用・修正・却下を判断
- 修正がある場合は Claude で再生成

### ステップ4: 投稿

- 承認済みの投稿をXに手動投稿
- 投稿時間は最適時間帯を狙う（後述）
- 投稿後、エンゲージメントを24時間追跡

### X最適投稿時間帯

| 時間帯 | 特徴 | おすすめ度 |
|---|---|---|
| 7:00〜8:00 | 通勤時間、情報収集モード | ★★★★☆ |
| 12:00〜13:00 | 昼休み、リラックスモード | ★★★★★ |
| 18:00〜19:00 | 帰宅時間、振り返りモード | ★★★★☆ |
| 21:00〜22:00 | 就寝前、内省モード | ★★★★★ |

## Threads投稿の半自動化フロー

### Threads の特徴（Xとの違い）

| 項目 | X (Twitter) | Threads |
|---|---|---|
| 文字数上限 | 280文字（日本語140文字相当） | 500文字 |
| トーン | 情報・速報・意見 | カジュアル・会話的 |
| ハッシュタグ | 重要 | タグ機能あり（控えめに） |
| アルゴリズム | フォロー+おすすめ | Instagram連携+おすすめ |
| 画像 | あると伸びる | あると大きく伸びる |

### ステップ1: X投稿をベースにThreads用にリライト

```
以下のX投稿をThreads向けにリライトしてください。

■ 元のX投稿:
{X投稿テキスト}

■ Threads向け調整ポイント:
- より会話的・カジュアルなトーンに
- 500文字まで使えるので、具体例やストーリーを追加
- 質問で終わり、コメントを促す
- ハッシュタグは最大3個に絞る
```

### ステップ2: 画像の検討

- Threads は画像付き投稿のリーチが大きい
- Canva や Figma で簡易ビジュアルを作成
- ブランドカラー（ミントグリーン `#6ecbb0`）を基調に

### ステップ3: 承認 → 投稿

- Xと同様に社長承認を経て投稿
- Instagram との連携投稿も検討

## 投稿テンプレート

### 共感系テンプレート（全体の70%）

**パターン1: 「あるある共感」型**
```
{ターゲットのあるある体験}

でも、それって{ポジティブな解釈}なんだよね。

{前向きな一言}
```

例:
```
「自分なんか」って思う日、ありませんか？

でもそれ、自分に期待してるからこそ感じる気持ちなんだよね。

今の自分を認めることが、次の一歩になる。
```

**パターン2: 「気づき共有」型**
```
最近気づいたこと。

{日常の中の小さな気づき}

{その気づきから得た学び}

{読者への問いかけ}
```

**パターン3: 「過去の自分へ」型**
```
{過去の状態}だった頃の自分に伝えたい。

{今だから言えるメッセージ}

同じように感じてる人がいたら、{励ましの一言}
```

---

### 情報系テンプレート（全体の20%）

**パターン1: 「数字で伝える」型**
```
知ってた？

{驚きの統計・データ}

つまり、{データの意味するところ}

{Rizupのミッションに繋がる一言}
```

**パターン2: 「3選・5選」型**
```
{テーマ}で大切なこと3つ

① {ポイント1}
② {ポイント2}
③ {ポイント3}

特に③は意外と見落としがち。
```

**パターン3: 「ステップ解説」型**
```
{悩み}を抜け出す3ステップ:

Step1: {行動}
Step2: {行動}
Step3: {行動}

まずはStep1だけでOK。
```

---

### 宣伝系テンプレート（全体の10%）

**パターン1: 「自然な紹介」型**
```
{共感できる課題}

そんな人たちが集まって、お互い応援し合える場所があったらいいなと思って、Rizupを作りました。

{アプリ/コミュニティの特徴を1つ}

興味ある方はプロフィールのリンクから覗いてみてください。
```

**パターン2: 「ユーザーの声」型**
```
Rizupを使ってくれている方から、こんな声をいただきました。

「{ユーザーの声（許可取得済み）}」

{感謝の一言と、アプリの価値}
```

**注意:** 宣伝系は週1〜2回に抑える。売り込み感を出さない。

## ハッシュタグ戦略

### X ハッシュタグ

**ブランドタグ（毎回付ける）:**
- `#Rizup`
- `#前向きに生きる`

**テーマ別タグ（2〜3個選択）:**

| カテゴリ | ハッシュタグ候補 |
|---|---|
| 自己成長 | `#自己成長` `#自分磨き` `#自己肯定感` `#メンタルヘルス` |
| キャリア | `#転職` `#副業` `#キャリアチェンジ` `#働き方` |
| マインド | `#ポジティブ思考` `#マインドセット` `#モチベーション` |
| コミュニティ | `#仲間募集` `#一緒に成長` `#朝活` |
| ライフスタイル | `#丁寧な暮らし` `#日記` `#振り返り` |

**運用ルール:**
- 1投稿あたり最大3個（多すぎるとスパム感）
- ブランドタグ1個 + テーマタグ2個が理想
- トレンドタグは無理に使わない（ブランドイメージ優先）

### Threads タグ

- Threads はタグ機能（トピック）を使用
- 1投稿1タグが推奨
- `自己成長` `メンタルヘルス` `キャリア` など大きめのトピックを選択

## エンゲージメント向上のコツ

### 1. 投稿の「型」を意識する

```
【伸びやすい型】
✓ 冒頭1行で興味を引く（「知ってた？」「正直に言うと」）
✓ 具体的なエピソードや数字を入れる
✓ 最後に問いかけで終わる（「あなたはどう思う？」）
✓ 改行を効果的に使う（読みやすさ重視）

【伸びにくい型】
✗ 抽象的なポエム調
✗ 長すぎる（Xは短いほど伸びる傾向）
✗ ネガティブで終わる
✗ ハッシュタグだらけ
```

### 2. リプライ・引用RTへの対応

- **30分以内の返信**を目標にする（アルゴリズム的に有利）
- お礼だけでなく、会話を続ける返信を心がける
- 批判的なリプライにも冷静に、ブランドボイスを維持
- 返信文も社長確認が必要な場合はテンプレートを用意

### 3. フォロワーとの関係構築

- ターゲット層のアカウントを毎日10件いいね
- 共感できる投稿には積極的にリプライ
- 定期的にアンケート機能を活用（「あなたはA派？B派？」）
- フォロワーの投稿を引用RTで紹介（許可取得の上）

### 4. 投稿パフォーマンスの振り返り

**週次チェック項目:**
- インプレッション数トップ3の投稿 → 共通点を分析
- エンゲージメント率（いいね+RT+リプ / インプレッション）
- フォロワー増減数
- プロフィールクリック数（コミュニティ導線の効果）

**月次KPI:**

| 指標 | 初月目標 | 3ヶ月目標 | 6ヶ月目標 |
|---|---|---|---|
| Xフォロワー | 300 | 1,000 | 3,000 |
| Threadsフォロワー | 100 | 500 | 1,500 |
| 平均インプレッション/投稿 | 500 | 2,000 | 5,000 |
| コミュニティ流入（月） | 10人 | 50人 | 150人 |

## ツール活用

### Claude AI を中心とした運用体制

Buffer や Hootsuite などの有料ツールを使わず、Claude AI + 手動投稿 で運用コストを最小化する。

### 運用フロー詳細

```
【毎週月曜・朝】週次コンテンツ準備
┌───────────────────────────────────────────────┐
│ 1. Kai のリサーチ結果を確認                      │
│ 2. 週のテーマ・キーワードを決定                   │
│ 3. Claude AI で14投稿（X）+ 5投稿（Threads）を一括生成 │
│ 4. /media/x-posts.md に保存                     │
│ 5. 社長承認                                     │
│ 6. 修正があれば Claude で再生成                   │
│ 7. 承認済み投稿に「✅」マークを付ける              │
└───────────────────────────────────────────────┘

【毎日・投稿時間】
┌───────────────────────────────────────────────┐
│ 1. 承認済み投稿を確認                            │
│ 2. X / Threads に手動投稿                       │
│ 3. 投稿URLを /media/x-posts.md に記録           │
│ 4. エンゲージメントを随時チェック                  │
└───────────────────────────────────────────────┘

【毎週金曜・夕方】週次振り返り
┌───────────────────────────────────────────────┐
│ 1. 週の投稿パフォーマンスを集計                   │
│ 2. トップ投稿の分析                              │
│ 3. 来週の改善ポイントを洗い出し                   │
│ 4. /media/content-calendar.md を更新            │
└───────────────────────────────────────────────┘
```

### Claude AI の活用シーン

| シーン | プロンプト例 |
|---|---|
| 投稿の一括生成 | 「来週のX投稿14本を生成して」 |
| リライト | 「この投稿をもっとカジュアルに」 |
| Threads変換 | 「このX投稿をThreads用に拡張して」 |
| パフォーマンス分析 | 「この3投稿の共通点を分析して」 |
| ハッシュタグ提案 | 「このテーマに合うタグを5つ提案して」 |
| 返信テンプレート | 「このリプライへの返信案を3つ」 |

### 将来的な自動化拡張

現時点では手動投稿だが、以下の段階で自動化を拡張予定:

**Phase 1（現在）:** Claude AI で下書き → 社長承認 → 手動投稿
**Phase 2:** X API + Threads API を取得し、承認済み投稿の予約投稿を実装
**Phase 3:** 投稿パフォーマンスの自動取得 → Claude AI が次週のコンテンツを最適化提案
**Phase 4:** 完全自動（下書き → 自動承認ルール → 予約投稿）※社長の最終判断は維持
