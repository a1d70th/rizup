# 不適切コンテンツ対策・即時制御システム 仕様書

---

## 1. リアルタイム自動検知

### フロー

```
ユーザーが投稿ボタンを押す
  ↓
Edge Function: /functions/v1/moderate
  ↓
Claude API（claude-haiku-4-5）に投稿内容を送信
  ↓
判定結果を返す
  ├── SAFE → 投稿をDBに保存 → タイムラインに表示
  ├── WARN → 投稿をブロック → Shoが優しく返す
  ├── CRISIS → 投稿をブロック → 相談窓口を即座に表示
  └── BAN → 投稿をブロック → アカウント即時停止
```

### 応答時間目標
- 判定完了まで：2秒以内
- ユーザー体験：投稿ボタン押下 → ローディング → 結果表示

## 2. 禁止コンテンツ分類

| カテゴリ | 内容 | 判定レベル |
|---|---|---|
| **誹謗中傷・個人攻撃** | 他ユーザーへの侮辱・人格否定 | WARN（1回目）→ BAN（繰り返し） |
| **自傷・自殺関連** | 自傷行為への言及・死にたい等 | CRISIS（即時介入） |
| **性的コンテンツ** | 性的な内容・わいせつ表現 | WARN |
| **差別・ヘイト** | 人種・性別・国籍等に対する差別 | WARN → BAN |
| **個人情報** | 電話番号・住所・メールアドレス | WARN（自動マスキング） |
| **広告・スパム** | 宣伝・勧誘・アフィリエイトリンク | WARN |
| **違法コンテンツ** | 違法薬物・犯罪教唆・児童搾取 | BAN（即時） |

## 3. モデレーション AI プロンプト

```
あなたはRizupアプリのコンテンツモデレーターです。
ユーザーの投稿を分析し、以下のJSON形式で判定を返してください。

判定レベル：
- "safe": 問題なし。投稿を許可
- "warn": 不適切な内容を含む。投稿をブロックし、書き直しを促す
- "crisis": 自傷・自殺関連。投稿をブロックし、相談窓口を表示
- "ban": 違法コンテンツ・深刻な誹謗中傷。即時アカウント停止

分析する観点：
1. 誹謗中傷・個人攻撃がないか
2. 自傷・自殺を示唆する表現がないか
3. 性的な内容がないか
4. 差別・ヘイト表現がないか
5. 個人情報（電話番号・住所・メール）が含まれていないか
6. 広告・スパム・勧誘がないか
7. 違法コンテンツがないか

重要なルール：
- ネガティブな感情の表現（「つらい」「悲しい」「うまくいかない」）は
  「safe」と判定してください。悩みの共有はRizupの正当な利用です。
- 「死にたい」「消えたい」「もう無理」等の表現は「crisis」と判定してください。
- 判定に迷う場合は「safe」側に倒してください（過剰なブロックを防ぐ）

出力フォーマット（JSONのみ）：
{
  "level": "safe" | "warn" | "crisis" | "ban",
  "reason": "判定理由（1行）",
  "category": "カテゴリ名",
  "detected_info": "検出した個人情報等（該当する場合）"
}

ユーザーの投稿：
{content}
```

## 5. 段階的ペナルティシステム

### ペナルティテーブル

| 違反回数 | 措置 | 復帰条件 |
|---|---|---|
| 1回目 | 警告 + 該当投稿を非表示 | 即時（書き直し可能） |
| 2回目 | 24時間の投稿停止 | 24時間経過後に自動解除 |
| 3回目 | 永久アカウント停止 | 不可（再登録も不可） |
| 即時停止 | 違法コンテンツ・深刻な誹謗中傷 | 不可 |

### 実装

```sql
-- ペナルティ管理テーブル
CREATE TABLE penalties (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID NOT NULL REFERENCES profiles(id),
  type        TEXT NOT NULL CHECK (type IN ('warn','block_24h','ban')),
  reason      TEXT NOT NULL,
  expires_at  TIMESTAMPTZ,  -- block_24h の場合のみ
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- 投稿前チェック関数
-- penalties テーブルを確認し、
-- ban → 投稿不可
-- block_24h & expires_at > NOW() → 投稿不可
-- warn 2回 → 次の違反で block_24h
-- block_24h 1回 → 次の違反で ban
```

## 6. 通報システム

### UI

- 全投稿の右上に `⋯` メニュー → 「通報する」ボタン
- 通報理由を選択：
  - 誹謗中傷・個人攻撃
  - 性的な内容
  - 個人情報の公開
  - スパム・広告
  - 自傷・自殺に関する内容
  - その他（自由記述）

### 処理フロー

```
ユーザーが通報
  ↓
reports テーブルに保存（status: 'pending'）
  ↓
運営に通知（メール or Slack Webhook）
  ↓
24時間以内に運営が確認
  ├── 問題なし → status: 'resolved'
  ├── 違反あり → 投稿非表示 + ペナルティ付与
  └── 深刻 → 即時非表示 + アカウント停止
```

### 自動エスカレーション

- 同一投稿への通報が3件以上 → 自動的に非表示 + 運営に即通知
- 同一ユーザーへの通報が5件以上/月 → 運営レビュー必須

## 7. 未成年保護

### 年齢確認フロー

```
サインアップ画面
  ↓
生年月日の入力（必須）
  ↓
年齢計算
  ├── 13歳未満 → 登録不可。「Rizupは13歳以上の方を対象としています」
  ├── 13〜17歳 → 「保護者の同意を得ていますか？」チェックボックス
  └── 18歳以上 → 通常フロー
```

### 未成年ユーザーへの追加保護

- プロフィールに年齢は表示しない
- DMは実装しない（グループのタイムラインのみ）
- 個人情報の入力を特に厳しくスキャン
