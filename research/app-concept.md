## 2. 既存サービスが失敗している理由

### 共通の失敗パターン

| # | 失敗理由 | 具体例 |
|---|---|---|
| 1 | **ネガティブスパイラル** | メンタルヘルス系SNSは悩みの共有が主になり、投稿がどんどん暗くなる。見ているだけでつらくなり離脱 |
| 2 | **継続の動機が弱い** | 日記アプリは記録するだけで終わり。「書いて何になるの？」と感じて3日で辞める |
| 3 | **ソーシャル要素の欠如** | セルフケアアプリは一人で完結する設計が多く、孤独感が解消されない |
| 4 | **フィードバックがない** | 記録しても誰からも反応がない。AIの分析もない。書きっぱなしになる |
| 5 | **ターゲットが広すぎる** | 全年齢・全属性を狙うとコミュニティの一体感が生まれない |
| 6 | **ゲーミフィケーション依存** | ポイントやバッジだけで継続させようとすると、新鮮さが失われた瞬間に離脱する |

### Rizup が解決する方法

| 失敗理由 | Rizup の解決策 |
|---|---|
| ネガティブスパイラル | 投稿のトーンを「前向き」に設計。AIが自動でポジティブリフレーミングを提案 |
| 継続の動機が弱い | AIが成長を可視化。「先月のあなたと比べて〇〇が変わりました」 |
| ソーシャル要素の欠如 | 仲間からのポジティブリアクション。毎日誰かが反応してくれる設計 |
| フィードバックがない | AIが毎日フィードバック + 仲間の応援。二重のフィードバックループ |
| ターゲットが広すぎる | 「20〜30代・夢があるけど自信がない」に絞る |
| ゲーミフィケーション依存 | ゲーム要素ではなく「本当の成長実感」で継続。AIが変化を言語化 |

## 4. Claude Code で作れる最初のバージョン設計

### 技術構成

```
フロントエンド：Next.js（React）+ Tailwind CSS
  → Claude Code で高速にUI構築可能
  → Vercel でデプロイ（既に利用中）

バックエンド：Supabase
  → 認証（メール / Google / Apple）
  → PostgreSQL データベース
  → リアルタイムサブスクリプション（タイムライン更新）
  → ストレージ（プロフィール画像）

AI機能：Claude API
  → 日記投稿へのフィードバック生成
  → 人物像分析（投稿蓄積後）
  → 毎朝の励ましメッセージ生成

プッシュ通知：Expo Push Notifications
  → 毎朝の励まし通知

ホスティング：Vercel
  → Next.js のデプロイ（既に利用中）
```

### データベース設計（Supabase）

```sql
-- ユーザー
users
  id          UUID PRIMARY KEY
  email       TEXT UNIQUE
  name        TEXT
  bio         TEXT
  goal        TEXT
  avatar_url  TEXT
  created_at  TIMESTAMP

-- 日記投稿
posts
  id          UUID PRIMARY KEY
  user_id     UUID REFERENCES users
  content     TEXT
  mood        INTEGER (1-5)
  ai_feedback TEXT
  created_at  TIMESTAMP

-- リアクション
reactions
  id          UUID PRIMARY KEY
  post_id     UUID REFERENCES posts
  user_id     UUID REFERENCES users
  type        TEXT ('cheer' | 'empathy' | 'amazing')
  created_at  TIMESTAMP

-- 励まし通知メッセージ
daily_messages
  id          UUID PRIMARY KEY
  content     TEXT
  date        DATE UNIQUE
  created_at  TIMESTAMP
```

### 開発ステップ（Claude Code で実行）

```
Week 1：基盤構築
  ├ Next.js プロジェクト作成
  ├ Supabase プロジェクト作成・DB設計
  ├ 認証（メールログイン）実装
  └ 基本UI（ログイン・プロフィール）

Week 2：コア機能
  ├ 日記投稿機能（テキスト + 気分）
  ├ タイムライン表示
  ├ Claude API でのAIフィードバック実装
  └ ポジティブリアクション機能

Week 3：成長可視化
  ├ 成長グラフ（気分推移・投稿日数）
  ├ 毎朝の励まし通知
  └ プロフィールページ

Week 4：仕上げ・テスト
  ├ UI/UXの磨き込み
  ├ レスポンシブ対応
  ├ βテスト（Discord メンバー向け）
  └ Vercel デプロイ・公開
```
